# Финальные концовки (`res://levels/cycles/level_end.tscn`)

## Что добавлено

- Новый скрипт уровня: `res://levels/cycles/level_end.gd`
- Новый скрипт холодильника (только для финального уровня): `res://objects/interactable/fridge/final_ending_fridge.gd`
- Новая финальная feeding-миниигра:
  - `res://levels/minigames/feeding/final_feed_minigame.tscn`
  - `res://levels/minigames/feeding/final_feed_minigame.gd`
- Новые сцены концовок и титров:
  - `res://levels/endings/good_ending_screen.tscn`
  - `res://levels/endings/bad_ending_screen.tscn`
  - `res://levels/endings/ending_credits.tscn`
  - `res://levels/endings/ending_screen.gd`
  - `res://levels/endings/ending_credits.gd`

## Логика ветвления

- Первый выбор игрока в `level_end` фиксирует ветку:
  - Ноутбук -> хорошая ветка.
  - Холодильник -> плохая ветка.
- После выбора одна из альтернатив отключается (без влияния на другие уровни):
  - при выборе ноутбука холодильник становится закрытым/неактивным;
  - при выборе холодильника ноутбук становится выключенным/неактивным.

## Хорошая концовка

1. Игрок проходит LLM-лабу на ноутбуке.
2. После завершения лабы включается кровать.
3. Сон переводит в `good_ending_screen.tscn`.
4. Далее автоматически запускаются титры.

## Плохая концовка

1. Игрок запускает специальную финальную feeding-миниигру.
2. Этапы еды:
   - пельмени;
   - другая еда;
   - торт;
   - рыба (временная реализация через суши-ассеты);
   - заглушки для будущих «неприятных» этапов.
3. Между этапами выполняется шейдерный глитч и ухудшается лицо Андрея.
4. На старте ветки планируется искажение квартиры с задержкой (по умолчанию 5 сек), чтобы оно не вспыхивало до входа в мини-игру.
5. Для `TextureDistortionManager` в `level_end` отключены `distortion_music` и `distortion_sfx` (без звука).
6. После завершения ветки запускается `bad_ending_screen.tscn`.
7. Если игрок умирает на этом уровне уже в ветке холодильника, вместо обычного death-screen тоже запускается плохая концовка.

## Кастомная смерть уровня

- В `GameDirector` добавлена точка расширения:
  - `handle_custom_death_screen()` у текущей сцены.
- Если сцена возвращает `true`, стандартный death-overlay не показывается.

## Музыка на будущее

- В титрах есть экспортируемые поля:
  - `credits_music`
  - `credits_music_volume_db`
  - `credits_music_fade_time`
- Можно назначить музыку позже прямо в `ending_credits.tscn`, не меняя код.
- В главном меню кнопка `Авторы` открывает ту же сцену титров (`ending_credits.tscn`), что и после концовок.

## Управление титрами

- Титры идут снизу вверх.
- Во время титров пауза отключена.
- `Esc`:
  - первое нажатие: показывает "Вы уверены, что хотите выйти? В следующий раз вам придётся проснуться снова...";
  - второе нажатие в течение короткого окна: досрочный выход в главное меню.
